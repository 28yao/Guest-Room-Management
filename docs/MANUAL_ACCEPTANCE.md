## 八、常见问题

| 现象 | 处理 |
|------|------|
| 表结构不一致（通用提示） | 见 README §数据库对照表；旧库按序补 **V3～V14**，或 `V1`+`V2` 全量建库 |
| 房态图报「表结构不一致」且 V3～V10 已执行 | 多为缺 **`room.clean_status`**，执行 **V14** 后重启后端 |
| 入住/换房报「表结构不一致」 | 入住常见缺 `folio_line.quantity`（**V9**）；换房重算常见缺 `folio.created_at`（**V10**）；看后端日志 `Unknown column` |
| 预订退订退款失败 | 检查 **V11**（`payment.folio_id` 允许 NULL） |
| 置脏后房态图丢预订/在住色 | 展示态应按**查看日**仍显示预订/在住；库内态可为脏房（弹窗区分展示/库内） |
| 房态图点击客房 | 日程：预订 **预订入住**、在住 **退房**/**退款**/**换房**；无占用可快速预订/Walk-in（须结账） |
| 房态图 Walk-in/预订入住「系统繁忙」 | 须传 `payments[]` 且已开班；收款=应付 |
| 23-24 在住后 24-25 无法入住 | 须先退房且空净；时刻默认 **18:00 入住 / 12:00 离店**，相邻两单间隔≥1h 可售 |
| 入住报「数据库连接失败」但 MySQL 正常 | 多为 **表字段与代码不一致**（如 `stay_order.guest_name` 未写入）；看后端日志 SQLException，执行 V8 |
| 办理入住页「系统繁忙」 | 可售查询时刻须为 ISO 格式 `yyyy-MM-ddTHH:mm:ss`；刷新前端后重试 |
| 登录 500 | 检查 `application-local.yml` |
| 前端 ECONNREFUSED | 先启动后端 |
| 房态图为空 | 执行 `sql/V4__room_seed.sql` 或手动建房 |
| **已授用户管理权仍提示无权限**（BUG-AUTH-01，**未修复**） | 属已知缺陷，**保留后期修改**。验收时用户管理相关用例（A-05～A-10b）请用 `admin` 执行；勿将前台授权后访问 `/system/users` 作为通过条件 |
