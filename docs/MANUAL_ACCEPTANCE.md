## 二、MOD-AUTH

### 2.1 登录 / 登出

| 编号 | 步骤 | 通过标准 |
|------|------|----------|
| A-01 | `admin` / `admin123` 登录 | 进入系统，默认 **房态图** |
| A-02 | 错误密码登录 | 失败提示，不进入系统 |
| A-03 | 退出登录 | 回到登录页 |
| A-04 | 登录后 F5 刷新 | 仍保持登录 |

### 2.2 用户管理

| 编号 | 步骤 | 通过标准 |
|------|------|----------|
| A-05 | 系统管理 → 用户管理，列表正常 | 无表结构报错 |
| A-06 | 新增用户 `test01` / `123456`，分配「前台」 | 列表出现 |
| A-07 | **编辑**：仅改状态/角色（对话框无密码项） | 保存成功 |
| A-08 | **修改密码**：点操作列「修改密码」，设新密码并确认 | 提示成功；用新密码可登录 |
| A-09 | 停用 `test01` 后尝试登录 | 无法登录 |
| A-10 | 重新启用后可登录 | 无系统管理菜单（仅前台权限时） |
| A-10a | 对 `test01` 点 **删除** 并确认 | 列表消失；提示成功 |
| A-10b | `admin` 与当前登录账号行 | 操作列无「删除」按钮 |

### 2.3 角色权限

| 编号 | 步骤 | 通过标准 |
|------|------|----------|
| A-11 | 系统管理 → 角色权限，选「前台」，改勾选后 **保存** | 再次打开勾选一致 |
| A-12 | 同角色点 **恢复默认**，确认 | 勾选恢复为 V2 种子默认（前台权限集） |
| A-13 | 选「系统管理员」→ **恢复默认** | 应为全部权限点 |

### 2.4 敏感权限直授

| 编号 | 步骤 | 通过标准 |
|------|------|----------|
| A-14 | 选用户，勾选 `billing:price:adjust` 等，**保存** | 保存成功 |
| A-15 | **恢复默认**，确认 | 全部直授清空（仅角色权限生效） |

### 2.5 权限隔离

| 编号 | 步骤 | 通过标准 |
|------|------|----------|
| A-16 | `test01` 访问 `/system/users` | 被拦截 |
| A-17 | 清除 token 后访问业务页 | 跳转登录 |

## 三、MOD-ROOM

> 建议用 `admin` 验收；导入 `V4__room_seed.sql` 后房态图约有 15 间演示房。

### 3.1 房型 / 客房

| 编号 | 步骤 | 通过标准 |
|------|------|----------|
| R-01 | 房型管理：新增/编辑门市价 | 列表更新 |
| R-02 | 客房管理：新增房号，重复房号 | 提示房号已存在 |
| R-03 | `test01` 无房型/客房管理菜单 | 菜单隐藏 |

### 3.2 房态图

| 编号 | 步骤 | 通过标准 |
|------|------|----------|
| R-04 | 房态图展示色块、房号、状态 | 正常 |

| R-11 | 无 `room:status:force` 的用户 | 无强制改态入口 |

## 五、常见问题

| 现象 | 处理 |
|------|------|
| 表结构不一致 | 执行 `sql/V3__auth_add_description.sql`、`sql/V5__schema_align_legacy.sql` 后重启后端 |
| 登录 500 | 检查 `application-local.yml` |
| 前端 ECONNREFUSED | 先启动后端 |
| 房态图为空 | 执行 `sql/V4__room_seed.sql` 或手动建房 |
| **已授用户管理权仍提示无权限**（BUG-AUTH-01，**未修复**） | 属已知缺陷，**保留后期修改**。验收时用户管理相关用例（A-05～A-10b）请用 `admin` 执行；勿将前台授权后访问 `/system/users` 作为通过条件 |
