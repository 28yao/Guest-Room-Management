# 文档先行原则

所有涉及文档修改、文件创建、代码迭代、功能调整和配置变更的任务，必须遵循“文档先行、同步更新”原则。

在执行具体变更前，应先核查相关文档是否完整、准确、最新，并与当前任务目标、实现逻辑和约束条件保持一致。若发现文档存在缺失、过时、冲突或逻辑不闭环的问题，必须先完成文档补全与修正，再进行实际修改或开发。

任何修改或创建类任务，都不得在文档未核验、未同步的情况下直接执行。任务完成后，应确保文档与实际结果一致，避免出现文档滞后、信息缺失或实现与说明不匹配的问题。

# 技术栈要求：

前端：Vue3 + Element Plus
后端：SpringBoot + MyBatis-Plus
数据库：MySQL


Java 所有类必须包含标准JavaDoc注释

```java
/**
* 类功能描述(说明该类核心职责)
*
* @author liuxinsi
* @date生成注释的时间规则:
*/
```


规则：

1.必须使用 JavaDoc 风格(/***/)
2.必须描述类的主要职责
3.禁止省略类注释
4.禁止使用嵌套循环
5.使用java8环境

Java 所有接口必须包含标准JavaDoc注释


需求database: 

 - 禁止生成DROP TABLE/DROP DATABASE语句 
 - 所有SQL必须使用参数化查询 
 - 操作前必须验证环境（生产/测试/开发） 
 - 危险操作必须添加二次确认机制
